\section{MDPs and Value Functions}

\begin{frame}{From Decisions to MDPs}
  \Large
  A Markov Decision Process adds dynamics and rewards:
  \[
    (\states,\actions,P,r,\gamma)
  \]

  \vspace{0.2cm}
  \normalsize
  \begin{itemize}
    \item $\states$: possible states
    \item $\actions$: possible actions
    \item $P(s' \mid s,a)$: transition
    \item $r(s,a)$: reward
    \item $\gamma \in [0,1)$: discount
  \end{itemize}

  \vspace{0.2cm}
  The key modeling assumption is that $s_t$ contains enough information to make the next decision.
  \note{States, actions, and policies were introduced earlier using the grain terminal example. Here, formalize the MDP by adding stochastic transitions, rewards, and discounting. Emphasize the Markov assumption: the state must contain enough information for decision making.}
\end{frame}

\begin{frame}{Return and Discounting}
  \Large
  Return is the total discounted reward from time $t$:

  \[
    G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k},
    \qquad 0 < \gamma < 1.
  \]

  \vspace{0.25cm}
  \normalsize
  The discount factor controls how farsighted the policy is.

  \vspace{0.25cm}
  \begin{itemize}
    \item small $\gamma$: more myopic, closer to optimizing the next railcar
    \item large $\gamma$: more weight on delayed consequences
    \item finance analogy: like net present value, rewards today count more than equal future rewards
    \item grain loading: a greedy policy optimizes each railcar; a farsighted policy optimizes the entire train
  \end{itemize}

  \note{Define return as the discounted sum of future rewards. Gamma is the planning horizon knob. Small gamma acts greedily; large gamma allows the policy to accept a locally suboptimal fill decision if it improves the full train, the contract, or later congestion.}
\end{frame}

\begin{frame}{$V$, $Q$, and Advantage}
  \begin{align*}
    V^\pi(s) &= \mathbb E_\pi[G_t \mid s_t=s] \\
    Q^\pi(s,a) &= \mathbb E_\pi[G_t \mid s_t=s,a_t=a] \\
    A^\pi(s,a) &= Q^\pi(s,a) - V^\pi(s)
  \end{align*}

  \vspace{0.3cm}
  \begin{itemize}
    \item $V$: how good is this state?
    \item $Q$: how good is this action in this state?
    \item $A$: was this action better than usual here?
  \end{itemize}

  \vspace{0.2cm}
  Advantage asks: was this action better or worse than the policy's usual action in the same state?
  \note{Advantage is the bridge to AWR, IQL, and IDQL. Make sure this concept lands.}
\end{frame}

\begin{frame}{Bellman Thinking}
  \Large
  \[
    Q(s,a) \approx r(s,a) + \gamma V(s')
  \]

  \vspace{0.4cm}
  \normalsize
  The value of an action is immediate reward plus the value of where it takes us.

  \vspace{0.4cm}
  Bellman recursion is the RL version of cost-to-go recursion in optimal control.
  \note{Connect explicitly back to dynamic programming. This equation is the skeleton underneath most of the algorithms later.}
\end{frame}

\begin{frame}{From Value to Action: Learn $Q(s,a)$}
  \Large
  If we know $Q(s,a)$, we can compare actions in the same state.

  \vspace{0.35cm}
  \[
    \text{choose the action with the highest predicted long-term value}
  \]

  \vspace{0.35cm}
  \normalsize
  This is why many RL algorithms learn an action-value function.
  \note{This slide bridges value functions to action selection. V tells us how good a state is; Q tells us which action to take.}
\end{frame}

\begin{frame}{Q-Learning Update}
  \Large
  A one-step target for $Q$:
  \[
    Q(s_t,a_t)
    \leftarrow
    r_t + \gamma \max_{a'} Q(s_{t+1},a')
  \]

  \vspace{0.3cm}
  \normalsize
  Immediate reward plus the best predicted value at the next state.
  \note{Introduce the core Q-learning backup. Do not dwell on stochastic approximation details. The important part is that the max turns value learning into policy improvement.}
\end{frame}

\begin{frame}{Greedy Policy Improvement}
  \Large
  Once we have $Q$, improve the policy by acting greedily:
  \[
    \pi_{\mathrm{new}}(s)
    =
    \arg\max_a Q(s,a)
  \]

  \vspace{0.35cm}
  \normalsize
  This is the basic RL loop:
  \[
    \text{estimate values} \rightarrow \text{choose better actions}.
  \]
  \note{Make the policy improvement idea explicit before discussing why it is dangerous offline.}
\end{frame}

\begin{frame}{Why Q-Learning Is Dangerous Offline}
  \Large
  Offline data only covers some actions.

  \vspace{0.35cm}
  \[
    \max_a Q(s,a)
  \]

  \vspace{0.25cm}
  \normalsize
  The max may select an action the dataset barely supports, where $Q$ is just a guess.

  \vspace{0.25cm}
  \takeaway{The same maximization that improves policies online can exploit value errors offline.}
  \note{This is the teaser for the offline RL section. Later, support mismatch and IQL will make this precise.}
\end{frame}
